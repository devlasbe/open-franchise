# Git 커밋 규칙

## 커밋 메시지 형식

```
<type>: <subject>

<body>

<footer>
```

### Type (필수)

| 타입 | 설명 | 예시 |
|------|------|------|
| `feat` | 새로운 기능 추가 | feat: 브랜드 검색 기능 추가 |
| `fix` | 버그 수정 | fix: 로그인 실패 시 에러 처리 |
| `refactor` | 코드 리팩토링 | refactor: BrandService 쿼리 최적화 |
| `chore` | 빌드/설정 변경 | chore: eslint 규칙 업데이트 |
| `docs` | 문서 수정 | docs: README 설치 방법 추가 |
| `test` | 테스트 추가/수정 | test: 브랜드 API 단위 테스트 추가 |
| `style` | 코드 포맷팅 | style: prettier 적용 |
| `perf` | 성능 개선 | perf: 이미지 지연 로딩 적용 |

### Subject (필수)

- 50자 이내
- 마침표 없이 작성
- 명령형 사용 ("추가" not "추가했음")
- 한글 또는 영어로 일관되게 작성

### Body (선택)

- 72자마다 줄바꿈
- "무엇을" 보다 "왜" 변경했는지 설명
- 여러 줄 가능

### Footer (선택)

- Breaking Change 명시
- 관련 이슈 참조

## 예시

### 기능 추가

```
feat: 브랜드 검색 자동완성 기능 추가

- 검색어 입력 시 실시간 추천 목록 표시
- 디바운스 적용으로 API 호출 최적화
- 키보드 네비게이션 지원

Closes #123
```

### 버그 수정

```
fix: 로그인 후 리다이렉트 오류 수정

이전 페이지 URL이 없는 경우 홈으로 이동하도록 수정

Fixes #456
```

### 리팩토링

```
refactor: 인증 로직 AuthService로 분리

- 컨트롤러에서 직접 처리하던 로직을 서비스로 이동
- 재사용성 및 테스트 용이성 향상
```

### 빌드/설정

```
chore: pnpm workspace 설정 추가

- pnpm-workspace.yaml 생성
- 루트 package.json에 워크스페이스 스크립트 추가
```

## 브랜치 전략

### 브랜치 네이밍

```
feature/브랜드-검색-기능
fix/로그인-버그
refactor/인증-로직-분리
```

### 작업 흐름

1. `main`에서 기능 브랜치 생성
2. 작업 완료 후 PR 생성
3. 코드 리뷰 후 머지

## 금지 사항

- `main` 브랜치에 직접 푸시 금지
- 의미 없는 커밋 메시지 금지 ("fix", "update", "asdf")
- 하나의 커밋에 여러 종류의 변경 포함 금지
- `.env` 파일 커밋 금지
